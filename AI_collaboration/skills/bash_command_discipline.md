description: 不要把启动服务、等待就绪、测试验证捆绑成一条长 Bash 链；三个阶段分步独立执行
tags: bash, 命令, 启动, 测试, 服务, background, 后台, 等待, 验证
---

需要启动一个服务并对其进行测试或验证时，把三个阶段拆成独立命令，不要用 `&&` 或 `;` 捆成一条长链：

1. **启动阶段**：以后台步骤单独启动长时间运行的服务（`run_in_background=true` 或 `& disown`），立即返回，不阻塞后续操作。
2. **等待阶段**：用有界轮询探测服务就绪（循环检查端口/健康端点，设定最大重试次数），不要用固定的长 `sleep`。
3. **测试/验证阶段**：服务就绪后，将每条测试或验证作为各自独立的命令执行，便于定位失败。

要点：
1. 阻塞/长驻进程不会返回，用 `&&` 串联后续步骤会使整条链永久挂住。
2. 固定 `sleep N` 容易过短（未就绪就测试）或过长（浪费时间）；有界轮询更健壮。
3. 命令粒度越小，输出越易归因，失败越易隔离。
4. 启动、等待、测试三步分开，任何一步失败都可单独重试，而不必重启整条流水线。

反例（不要这样做）：
```bash
python server.py & sleep 5 && curl http://localhost:8000/health && pytest tests/
```

正例（应该这样做）：
```bash
# 步骤 1：后台启动
python server.py &

# 步骤 2：有界轮询等待就绪（最多重试 20 次）
for i in $(seq 1 20); do
  curl -sf http://localhost:8000/health && break
  sleep 1
done

# 步骤 3：独立执行测试
pytest tests/
```
